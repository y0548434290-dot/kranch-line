# n8n Workflow For Order Summary Emails

הקובץ המעודכן לייבוא ל־n8n הוא:

- [Kranch - Google Sheets Order Email Summary.json](C:/vscode/yemot-system/docs/Kranch%20-%20Google%20Sheets%20Order%20Email%20Summary.json)

המטרה שלו:

- כל דקה לקרוא את גיליון `הזמנות`
- לחפש שורות שיש בהן ערך בעמודת `מייל`
- לשלוח מייל סיכום מעוצב
- לסמן בגיליון שהמייל כבר נשלח כדי לא לשלוח שוב

## לפני הייבוא

כדאי להוסיף לשיטס עוד 2 עמודות חדשות בסוף הטבלה:

- `אישור מייל נשלח`
- `סטטוס שליחת מייל`

הוורקפלואו ישתמש בהן כדי לדעת אם כבר נשלח מייל.

## איך לייבא ל־n8n

1. פתחי את `n8n`
2. בחרי `Import from file`
3. טעני את הקובץ `docs/Kranch - Google Sheets Order Email Summary.json`

## מה לעדכן אחרי הייבוא

1. ב־`Read Orders Sheet`
החליפי את `PUT_SPREADSHEET_ID_HERE` ב־`Spreadsheet ID` של השיטס שלך.

2. ב־`Mark Row As Sent`
החליפי גם שם את `PUT_SPREADSHEET_ID_HERE`.

3. חברי `Google Sheets credentials`
הוורקפלואו צריך גישת Google Sheets.

4. חברי `Gmail credentials`
השליחה כרגע מוגדרת דרך Gmail מתוך n8n, כדי שלא תצטרכי דומיין מאומת ב־Resend.

## איך זה עובד

הוורקפלואו מחפש שורות שבהן:

- יש ערך בעמודת `מייל`
- עמודת `אישור מייל נשלח` ריקה

אם שני התנאים מתקיימים:

1. נבנה HTML מעוצב מתוך נתוני ההזמנה
2. נשלח מייל ללקוחה
3. השורה תעודכן עם תאריך שליחה וסטטוס `נשלח`

## הערה חשובה

ב־n8n לפעמים יש הבדלים קטנים בין גרסאות של ה־Google Sheets node וה־Gmail node. אם אחרי הייבוא אחד השדות לא ממופה אוטומטית, פשוט פתחי את ה־node ומפי אותו מחדש דרך הממשק.

אם תרצי, השלב הבא אני יכול להכין לך גם גרסת `Make.com` במקום `n8n`, או לעזור לך להתאים את הבלופרינט הזה כך שישלח רק על שורה חדשה ולא על כל סריקה.
